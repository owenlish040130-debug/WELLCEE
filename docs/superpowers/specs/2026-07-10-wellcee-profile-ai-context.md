# Wellcee 个人资料完善流程 —— AI 增强设计 · 项目背景

> 创建日期：2026-07-10
> 状态：头脑风暴完成，待方案比选

## 一、Wellcee 产品概览

| 维度 | 说明 |
|------|------|
| 中文名 | 唯心所寓 |
| 定位 | 无中介 C2C 个人直租 + 国际化租房社交平台 |
| 创立 | 2017 年，上海 |
| 覆盖 | 国内 40+ 城市，用户来自全球 170+ 国家 |
| 核心用户 | 留学生、外籍租客、年轻白领 |
| 语言 | 中英多语言 |

**核心业务**：
- 三大真实房源：个人房东直租、租客个人转租、合租招室友
- 双向实名认证 + 房源审核 + 中介屏蔽机制

**差异化特色**：
- 租房 + 青年社交社区双轮驱动
- 智能室友匹配（作息/养宠/爱好/生活习惯标签）
- 同城生活板块（闲置、宠物领养、线下活动）
- 国际化互助社区

---

## 二、项目目标

重新设计 Wellcee「个人资料完善流程」，结合 AI 大模型：
- 让用户更便捷地完善个人资料
- 提升资料完整度和个性化表达
- 数据产出可服务于找房推荐、室友匹配、信任建设

**场景**：登录后 → 我的 → 个人主页 → 编辑资料

## 三、已确定的设计决策

### 3.1 AI 核心定位

- **效率 + 深度并重**：既要填得快，也要表达得准
- **渐进式混合型交互**：保留传统表单作为基础入口，AI 作为可选增强层
- **用户主动唤起 AI**：AI 不自动插话，等用户点击「AI 帮我看看」才工作

### 3.2 固定表单字段（7 个）

| 字段 | 类型 | 说明 |
|------|------|------|
| 头像 | 图片上传 | 信任建设核心 |
| 昵称 | 文本输入 | 基础身份 |
| 年龄 | 出生年份选择器 | 替代"心理年龄"，同时为五行计算提供输入 |
| 职业 | 文本输入 | 信任信号 + 室友画像 |
| 地区 | 选择/输入 | 国际化核心字段 |
| 语言 | 多选添加 | 国际化匹配基础 |
| 当前需求 | 多选卡片 | 出租 / 转租 / 找室友（有房/无房）/ 暂无需求 |

### 3.3 需求栏设计（不做硬性身份切割）

用户可多选，不强制二选一「房东/租客」身份：
- 🏠 我要出租（整租 / 单间）
- 🔄 我要转租（整租转租 / 单间转租）
- 🤝 我要找室友（已有房 / 还没房）
- 👀 暂无需求

### 3.4 标签体系（三层）

| 层级 | 标签类型 | 提取范围 |
|------|---------|---------|
| 通用标签 | 性格标签（MBTI、安静/外向等）、爱好标签（运动、宅家、社交等） | 所有人 |
| 需求标签 | 室友期望（作息、清洁、社交频率等）、求租偏好（区域、预算、房型等） | 按需求栏触发 |
| 房源标签 | 出租/转租房源特征（位置、户型、装修等）、转租原因 | 选出租/转租时触发 |

### 3.5 AI 追问机制

- **触发**：用户主动点击「AI 帮我看看」
- **策略**：具象化追问 / 场景化追问 / 情感化追问 / 画像补充追问
- **退出**：单条跳过、本轮全跳、连续 3 轮全跳后自动降级为静默模式
- **限制**：每次 1-3 条建议，追问必须附带选项

### 3.6 MBTI 处理

- 不放在固定表单中
- AI 不从文本推断
- 用户在自由文本中提到了 → 直接提取为标签
- 没提到 → 追问环节询问（附带简短说明帮助不知道 MBTI 的用户选择）

### 3.7 五行处理

- 从出生年份自动计算，不增加用户填写负担
- 标注为"文化特色"选填字段

### 3.8 差异化输入引导

根据需求栏动态切换自由文本输入引导文案：

| 用户需求 | 引导文案 |
|---------|---------|
| 暂无需求 | 介绍你自己——性格、爱好、生活方式... |
| 找室友 | 介绍你自己 + 理想室友画像 |
| 求租 | 介绍你自己 + 想找的房子描述 |
| 出租 | 介绍你自己 + 房源描述 |
| 转租 | 介绍你自己 + 转租原因 + 房源描述 |
| 多选 | 拼接组合（自我介绍 + 需求描述） |

### 3.9 信任建设

- 一期：匹配透明度（浏览者视角展示共同点/差异点高亮）—— 二期实现

## 四、完整用户流程（已确认）

```
Step 1: 固定表单填写
    ├─ 头像、昵称、年龄、职业、地区、语言
    └─ 当前需求（多选卡片）
        ↓
Step 2: 自由文本输入
    ├─ 引导文案根据需求栏动态切换
    └─ 支持文字 / 语音输入
        ↓
Step 3: AI 追问（用户主动点击「AI 帮我看看」唤起）
    ├─ 追问方向根据需求栏动态调整
    ├─ 每条可单独跳过
    └─ 每轮可全部跳过
        ↓
Step 4: AI 提取标签
    ├─ 通用标签（性格 + 爱好）—— 所有人
    ├─ 需求标签（室友期望 / 求租偏好 / 房源特征）—— 按需
    └─ 标签自动分类展示
        ↓
Step 5: 用户确认 / 调整 / 补充标签
        ↓
Step 6: AI 生成个人简介
        ↓
Step 7: 完成
    （未来：匹配透明度 - 浏览者视角共同点/差异点）
```

## 五、技术约束与假设

- 项目从零开始，无现有代码约束
- AI 大模型：假设使用 Claude API 或等效 LLM
- 语音输入为可选项（需语音识别能力）
- 五行计算为后端固定算法（非 AI）
- 一期不含匹配引擎改造
